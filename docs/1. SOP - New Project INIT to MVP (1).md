# **SOP \- AI Application Roadmap** 

Objective: Build a scalable, production-grade SaaS using "Infrastructure-as-Code" and "Content-as-Code" principles.

Method: AI Agents (Strategy/Asset Gen) \+ Human (Infra Setup) \+ IDE (Execution).

## **🛠️ The Technical Stack**

* **Framework:** Next.js 14+ (App Router, TypeScript)  
* **Database:** Neon (Serverless Postgres) via Prisma/Drizzle  
* **Hosting:** Vercel (Edge Network)  
* **Auth:** Clerk  
* **Payments:** Stripe  
* **Email:** Resend (React Email)  
* **UI/Styling:** Tailwind CSS, Shadcn/UI  
* **Media:** DALL-E 3 / Midjourney (Images), ElevenLabs (TTS)  
* **Validation:** Zod  
* **Observability:** Sentry, PostHog, Vercel Analytics

## **Phase 2: Setup & Infrastructure (Integration-First)**

**1\. Local Setup**

* Create project folder, docs/ folder (add generated files), and scripts/ folder.

**2\. Domain & DNS ([Cloudflare](https://www.cloudflare.com/))**

* Register Domain in Cloudflare.

**3\. Repository & Hosting ([Vercel](https://vercel.com/))**

* **GitHub:** Create empty repo.  
* **Vercel:** Create Project $\\rightarrow$ Import GitHub $\\rightarrow$ Add Domain $\\rightarrow$ Verify via Cloudflare.

**4\. Database (Neon via Vercel)**

* **Vercel Storage:** Select Neon $\\rightarrow$ Create Database.  
  * *Result:* Env Vars (POSTGRES\_URL, etc.) are auto-injected.

**5\. Connectivity (Resend & Keys)**

* **Resend:** Add Domain $\\rightarrow$ Verify via Cloudflare (Auto-DNS) $\\rightarrow$ Add RESEND\_API\_KEY to Vercel.  
* **Clerk/Stripe:** Create Accounts $\\rightarrow$ Add Keys to Vercel Env Vars.

**6\. 🛡️ Cost & Safety Controls (Critical)**

* **OpenAI/Anthropic:** Set "Hard Billing Limit" ($50/mo).  
* **Vercel:** Enable "Spend Management" (Pause project if limits exceeded).

Add ./scripts/[dev.sh](http://dev.sh) file to run locally on a unique port clearing the port first

## **Phase 3: Execution & Coding (The Consumption Phase)**

**Prompt Antigravity/Cursor:** "I am the Lead Architect. Follow this strict execution order. Do not proceed to the next step until the current one is verified."

### **Step A: Ingest & Configure**

1. **Context:** Read all files in docs/.  
2. **Sync:** Run vercel env pull .env.local to get the live Production/Preview keys.  
3. **Config:** Setup next.config.js and package.json.

### **Step B: Asset Fabrication (Make the Parts)**

* **Branding:** Apply 08\_Branding\_Guidelines.md to tailwind.config.ts and globals.css.  
* **Media:** Execute prompts in 09\_Media\_Gen\_List.md. Save optimized WebP files to public/.  
* **Legal:** Convert 13\_Legal\_Docs.md to TSX pages (/privacy).  
* **Emails:** Build React Email components from 14\_Email\_Templates.md.

### **Step C: Backend & Logic**

* **Database:** Create schema.prisma from 07\_DB\_Schema.md. Run npx prisma db push.  
* **Webhooks:** Build /api/webhooks/stripe and /api/webhooks/clerk (sync user data to Neon).  
* **Services:** Implement business logic from 11\_Business\_Logic.md.

### **Step D: Frontend Experience**

* **Pages:** Build Landing Page (Doc 04\) and App Dashboard (Doc 05).  
* **Notifications:** Implement the system from 12\_Notifications.md (Toast \+ Email triggers).

### **Step E: Hardening & Compliance (Security)**

* **Rate Limiting:** Add Upstash/Arcjet middleware to API routes.  
* **Consent:** Add CookieConsent.tsx component (Blocks Analytics until accepted).  
* **SEO:** Dynamic Sitemap (from DB), robots.txt, and JSON-LD Schema.

## **Phase 4: Content Engine (The Blog)**

* **Strategy:** Gemini generates the **Content Strategy Table** (Topic, Keyword, Slug, Hook).  
* **Build:** IDE generates content/blog/\[slug\].mdx with:  
  * Sticky Table of Contents.  
  * "Written by AI \+ Verified by Team" note.  
  * Newsletter Signup Form.  
  * Related Posts logic.  
* **Distribution:** IDE generates marketing/\[slug\]-social-pack.md (Twitter Thread, LinkedIn Post, Email Blast).

## **Phase 5: Final Validation & Handover**

**Use Gemini/Grok to:**

1. **Audit vs PRD:** "Does the deployed app meet all requirements in 10\_PRD.md?"  
2. **Visual QA:** "Check public/images \- are all assets generated and used?"
